# Security Policy

## Supported Maturity

This repository is currently a scaffold and not yet a hardened production release. Treat current implementations as pre-production.

## Reporting a Vulnerability

- Do not disclose sensitive issues in public issues or discussions.
- Send a private report to the maintainers with:
  - Affected component(s)
  - Reproduction steps
  - Potential impact
  - Suggested remediation (if known)

Until a dedicated security contact address is configured, open a minimal private channel with project maintainers before sharing technical details broadly.

## Security Principles

- Local-first operation whenever possible.
- Explicit confirmation before potentially destructive actions.
- Dry-run support for meaningful changes.
- Evidence preservation before remediation.
- Clear separation between local evidence and external LLM escalation.
